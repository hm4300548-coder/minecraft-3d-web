🔧 API Endpoints (جاهزة مباشرة):
════════════════════════════════════════════════════════════════════════════════

GET  /api/health                          ← فحص حالة السيرفر
GET  /api/blocks/load?userId=default      ← تحميل جميع الكتل
POST /api/blocks/save                     ← حفظ كتلة واحدة
POST /api/blocks/save-batch               ← حفظ عدة كتل
DELETE /api/blocks/delete                 ← حذف كتلة
DELETE /api/blocks/clear?userId=default   ← حذف جميع الكتل
GET  /api/blocks/stats?userId=default     ← الإحصائيات
GET  /api/blocks/range?...                ← كتل في منطقة

🎯 تدفق البيانات:
════════════════════════════════════════════════════════════════════════════════

عند البناء (Right Click):
  ──────────────────────────────────────────────────────────
  User Click
    ↓
  placeBlock() في BlockInteraction.js
    ↓
  world.setBlock() ← إضافة محلية
    ↓
  blockStorage.saveBlock() ← إرسال للسيرفر
    ↓
  Server: POST /api/blocks/save
    ↓
  Supabase Database ← محفوظ بشكل دائم! ✅

عند التحميل:
  ──────────────────────────────────────────────────────────
  Page Load
    ↓
  main.init() → loadSavedBlocks()
    ↓
  blockStorage.loadBlocks()
    ↓
  Server: GET /api/blocks/load
    ↓
  Supabase Database ← استرجاع البيانات
    ↓
  world.setBlock() لكل كتلة
    ↓
  ✅ عالم مكتمل جاهز!

  - عند البناء → blockStorage.saveBlock()
  - عند الهدم → blockStorage.deleteBlock()
  - عند الفتح → loadSavedBlocks()

يدوياً من Console:
  const bs = window.game.getBlockStorage();

  // حفظ
  await bs.saveBlock(0, 64, 0, 'STONE');

  // تحميل
  const blocks = await bs.loadBlocks();

  // حذف
  await bs.deleteBlock(0, 64, 0);

  // إحصائيات
  const stats = await bs.getStats();

🐛 حل المشاكل السريع:
════════════════════════════════════════════════════════════════════════════════

❌ "npm: command not found"
  → ثبت Node.js من nodejs.org

❌ "Port 3000 in use"
  → غيّر PORT في .env إلى رقم آخر

❌ "Cannot connect to database"
  → تحقق من SUPABASE_URL و SUPABASE_ANON_KEY

❌ "Blocks not saving"
  → افتح Console (F12) وابحث عن الأخطاء
